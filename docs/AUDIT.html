Setiap issue di-grade by demo damage. Critical = juri instant suspicion. High = credibility leak. Medium = polish.

#1 — Company profile bocor mock data (London, UK · 51-200 employees)
CRITICAL FIXED

Apa yang terjadi: waktu live mode, cockpit nampilin Ramp HQ di "London, UK" dengan size "51-200 employees". Padahal Ramp HQ di New York dan punya ~1500 karyawan. Data ini dari random pick di mock_data.py — nggak di-override sama live evidence.

Damage: juri yang familiar Ramp langsung tahu ini bohong. Kredibilitas anjlok.

Fix: _scrub_profile_for_live() — di live mode, set industry/hq/size/description = None. Frontend conditional rendering otomatis hide field-field itu.

Before{ "hq": "London, UK", "size": "51-200", "description": "...[DEMO DATA — not verified]" }
After (live mode){ "name": "Ramp", "domain": "ramp.com", "hq": null, "size": null, "description": null }
#2 — Action Pack bocor placeholder template {first_name}
CRITICAL FIXED

Apa yang terjadi: cold email di Action Pack masih nampilin Hi {first_name} dan — {your_name}. Plus sales angles nyebut "G2 pain", "competitor discount", "cloud partnership" — semua mock data yang nggak ada di live evidence.

Damage: instant "AI ini fake" reaction dari juri. Placeholder yang nggak ke-render = looks like a bug.

Fix: dua-layer. (1) MiMo prompt di-tighten supaya return action_pack overlay yang reference real numbers. (2) Validation: kalau MiMo return placeholder, REJECT — fallback ke _live_action_pack() yang build deterministic dari live evidence dengan placeholder square-bracket [Name] (intentional, bukan broken).

BeforeHi {first_name}, Noticed Ramp is leaning into enterprise — new tier, fresh hiring across Sales and RevOps, and a recent cloud partnership. — {your_name}
After (Ramp)Hi [Name], Congrats on the recent $300M raise and reaching a $32B valuation. Doubling revenue and customers in a year is a massive operational lift. — [Your Name]
#3 — Web Unlocker text dump berisi nav junk + boilerplate
HIGH FIXED

Apa yang terjadi: extracted article text dari PR Newswire kelihatan: "...Ramp Raises $500M... Accessibility Statement Skip Navigation Resources . Investor Relations . Journalists . Send a Release ..."

Damage: evidence row Web Unlocker keliatan sloppy. Bukan article — keliatan kayak HTML dump.

Fix: upgrade _strip_html_to_text(): (1) drop <nav>/<header>/<footer>/<aside> blocks, (2) prefer <article> / <main> / common post-body containers, (3) regex filter: drop lines < 25 chars OR matching menu patterns (Accessibility, Privacy, Cookies, Skip to, dll.).

BeforeRamp Raises $500 Million... Accessibility Statement Skip Navigation Resources . Investor Relations . Journalists . Agencies . Client Login . Send a Release . News . Products . Contact . Search...
AfterRamp Reaches $32 Billion Valuation, Doubling Revenue and Customers in Past Year Part of $300M Raise and Employee Tender Offer NEW YORK, Nov. 17, 2025 /PRNewswire/ -- Ramp, the leading financial operations platform...
#4 — Why-Now Reason terlalu verbose, hedgy, no concrete numbers
HIGH FIXED

#5 — Mock signals + Web Scraper [DEMO] evidence leaking ke Evidence Ledger
HIGH FIXED

Apa yang terjadi: EvidenceLedger nampilin row dengan tag [DEMO] saat mode live. Konflik visual: badge atas bilang LIVE, badge row bilang demo.

Fix: step 4a baru di _build_response: kalau ada minimal 1 live evidence, drop semua mock evidence + mock signals. Signal cards di-regenerate dari live data via _inject_live_signals().

#6 — Bright Data SERP response parser nggak nge-baca body
CRITICAL FIXED

Apa yang terjadi: Bright Data SERP API balas HTTP 200 tapi parser kita selalu return empty. Status di infra log fallback tapi latency real (~2.5s).

Root cause: Bright Data wrap response dalam struktur {status_code, headers, body: "<JSON string>"}. Parser kita langsung baca raw["organic"] di top-level — sebenarnya datanya nested di raw["body"] yang harus di-parse JSON ulang.

Fix: tambah _unwrap_serp_body() helper. Parser sekarang detect kalau ada field body: str → json.loads() dulu. Sekali ini di-fix, evidence count langsung naik dari 0 → 7 per call.

#7 — MiMo model name salah default (MiMo-V2.5 → 400 error)
HIGH FIXED

Apa yang terjadi: default model name MiMo-V2.5 ditolak Xiaomi API dengan "Not supported model". Probe ke endpoint /v1/models revealed lowercase format: mimo-v2.5.

Fix: default model di mimo_service.py diganti ke mimo-v2.5 (lowercase + dash). Sekarang synthesis jalan, executive_summary + why_now_reason + action_pack semua keluar real.

#8 — Confidence score stuck di "medium" karena hiring + review signal kosong
MEDIUM OPEN · acceptable

Apa yang terjadi: Why-Now confidence selalu "medium" (jarang "high") karena scoring engine cari 6 signal kind (hiring, news momentum, product launch, competitor pressure, review pain, funding/partnership). Hiring + review datanya cuma dari Web Scraper API yang masih di mode architecture.

Status: NOT fixed, dan aman dibiarkan. Medium confidence justru jujur — kita udah explicit di UI bilang Web Scraper "wired · not active". Kalau confidence "high" tapi data partial, itu over-promise.

Demo angle: bisa jadi talking point: "Notice the confidence is medium — we don't have hiring or review data this run. With Scraper dataset wired, confidence rises to high."
