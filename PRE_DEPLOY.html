04 Deploy Checklist (Vercel + Render)

Step-by-step. Estimate: 2-3 jam end-to-end.

Backend → Render (atau Fly.io)

  • Bikin backend/Dockerfile kalau belum (Python 3.11 + uvicorn)
  • Push repo ke GitHub kalau belum (private OK, public boost)
  • Render → New Web Service → connect repo, build root = backend/
  • Set env vars di Render dashboard: BRIGHT_DATA_API_TOKEN, BRIGHT_DATA_SERP_ZONE=serp_api1, BRIGHT_DATA_UNLOCKER_ZONE=web_unlocker1, MIMO_API_KEY, MIMO_BASE_URL=https://api.xiaomimimo.com/v1, MIMO_MODEL=mimo-v2.5, USE_MOCK=false
  • Upload backend/snapshots/*.json via commit (atau persistent disk Render)
  • Setelah deploy, smoke test: curl https://<render-url>/health
  • Pre-warm: curl -X POST https://<render-url>/warmup

Frontend → Vercel

  • Vercel → New Project → import repo, root = frontend/
  • Set env var: NEXT_PUBLIC_API_BASE=https://<render-url>
  • Deploy. Catat URL (akan jadi link submission).
  • Test analyze NVIDIA dari public URL — pastikan SSE timeline stream OK

  1. "Deterministic scoring, not LLM hallucination" — main differentiator. 90% kompetitor pakai LLM untuk angka. Lo nggak.

  3. "Every claim has a source URL" — pas klik evidence row, scroll ke source. Visual proof.
  4. "Pre-warmed snapshot architecture" — kalau ditanya soal Web Scraper latency.
  5. "Multi-provider LLM cascade with failover" — kalau ditanya soal resilience. Claude → MiMo → mock.
