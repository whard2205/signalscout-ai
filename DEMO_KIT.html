P7 Judge Objection Handling — Top 8 Tough Q&A

Pre-written responses untuk pertanyaan paling sulit yang bisa muncul dari judges. Hafalin core phrase tiap row.

If judge asks…Your response (CORE PHRASE in bold)
"Why use Python for scoring, not Claude?" Because LLMs hallucinate numbers. Our weights come from Trigger Event Selling, 6sense, Bombora, and Forrester frameworks — published B2B GTM research. Sales ops can audit scoring.py and tune for their pipeline. You can't audit a hallucination.
"Where's the proof the LLM is actually working?" (klik exec summary) See the badge: "Synthesized by Claude · evidence-grounded". This paragraph references 9 specific evidence rows below. The Why-Now score has a separate badge: "deterministic, not LLM". Two code paths: LLM for context, code for math.
"How do I know these weights are right?" You don't, and we don't claim the exact numbers are universally calibrated. The ORDERING is informed by common B2B trigger-event and intent-data frameworks (Trigger Event Selling by Craig Elias, 6sense buying stages, Forrester Demand Unit Waterfall). The VALUES are tunable defaults exposed in code so sales-ops can adjust them to their own pipeline data. This is a transparent heuristic, not a prediction model. No black box.
"Why is every company scoring 100?" They're not anymore — Amazon=99, NVIDIA=82, Marriott=71, Affirm=75, unknown companies score 30-45. Companies with funding + hiring + product + expansion all positive will saturate near 100 — that's the point: they're objectively prime outreach targets. Try a no-signal company to see the floor.
"What about Multi-Agent frameworks like CrewAI?" We use 7 named agents visible in the live timeline — Discovery, Mapping, Hiring Intel, Deep Read, Orchestration, Scoring, Synthesis. Plus we ship a real MCP server at /mcp (JSON-RPC 2.0) — any Claude Desktop, ChatGPT plugin, or MCP client can call our tools directly. CrewAI is a framework; we ship a protocol.
"Show me your MCP integration" (open terminal) curl -X POST http://<backend>/mcp -d '{"jsonrpc":"2.0","method":"tools/list"}' — returns 3 tools: analyze_company, get_evidence, compare_companies. Spec-compliant Anthropic MCP 2024-11-05. Same protocol Bright Data's own MCP server uses. Verifiable, not aspirational.
"Is the data really live or pre-fetched?" Mix — and that's production-grade. SERP × 2 and Web Unlocker fire live on every request (~7 seconds total). Web Scraper API runs at warmup time and serves cached snapshots — that's how real enterprise sales tools work. Click the reproducibility hash at the bottom of Evidence Ledger to verify determinism.
"How is this different from [competitor product]?" Three differentiators: (1) Deterministic scoring with research citations — competitors hallucinate numbers. (2) Evidence ledger with per-row source URLs and tier badges. (3) Multi-provider LLM cascade with mock-mode resilience — Claude fails? MiMo takes over. Network dies? Mock mode renders. Production-grade resilience.
"What's the business value?" Sales reps waste ~30% of their week on "why-now" research. Average SDR makes $60K base, so that's ~$18K/year in research time per rep. For a 50-rep team: $900K/year saved. Plus better cold-email response rates when timing is right. One signal, two ROI vectors.
