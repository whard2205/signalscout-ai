01 Apa Sistem Ini?

SignalScout AI menjawab satu pertanyaan untuk tim sales/GTM: “Kenapa kita harus contact company ini sekarang?” Output-nya bukan chatbot, tapi intelligence cockpit — satu layar dashboard yang menampilkan skor Why-Now, signal grid, evidence ledger, action pack (cold email + LinkedIn + discovery), competitor table, dan agent timeline live.

Target Optimasi

Menang demo live 3 menit. Wow factor + clarity untuk juri > kedalaman teknis.

Prinsip No-BS

Setiap klaim wajib punya evidence ID + source + Bright Data tool + confidence. Tidak ada halusinasi.

Mode Aman

Mock mode selalu jalan. Kalau API live gagal di tengah demo, cockpit tetap render lengkap.

SSE — endpoint streaming

Untuk efek timeline live, /analyze/stream emit event:

// Urutan event SSE
start   → cockpit menampilkan indikator "Live"
step    → tandai step current = done, advance step berikutnya = running
result  → render laporan lengkap
end     → tutup stream, set running = false
Catatan sinkronisasi: TIMELINE_STEPS di backend/app/api/routes.py dan PIPELINE di Cockpit.tsx dijaga sinkron by key. Kalau nambah step, tambahin di kedua tempat dengan key yang sama.

04 Setup & Install

A. Backend (Python 3.11+)

# di folder proyek
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1    # PowerShell Windows
pip install -r requirements.txt
copy .env.example .env       # biarkan USE_MOCK=true dulu

B. Frontend (Node 18+ / 20+)

cd frontend
npm install
copy .env.local.example .env.local
Tanpa API key pun bisa jalan. Default USE_MOCK=true + tidak ada ANTHROPIC_API_KEY = mode mock. Cockpit tetap menampilkan laporan lengkap dengan data seeded yang konsisten.

05 Cara Menjalankan

Buka dua terminal, biarkan keduanya hidup selama demo.

Terminal 1 — Backend

cd backend
.\.venv\Scripts\Activate.ps1
uvicorn main:app --reload --port 8000

Akan listen di http://localhost:8000

Terminal 2 — Frontend

cd frontend
npm run dev

Akan listen di http://localhost:3000

06 Cara Tes & Verifikasi

Tes bertahap dari paling dasar (backend hidup) sampai paling visual (cockpit lengkap). Kalau step gagal, jangan lanjut.

  • ✓
    Step 1 — Backend hidup. Buka http://localhost:8000/health. Harus return JSON seperti {"status":"ok","mode":"mock","claude":"mock"}. Kalau error → cek terminal backend.
  • ✓
    Step 2 — Endpoint /analyze. Buka Swagger UI di http://localhost:8000/docs. POST /analyze dengan body {"company":"Ramp"}. Harus return JSON berisi scores, signals, evidence, action_pack, dan mode:"mock".
  • ✓
    Step 3 — Cockpit render. Buka http://localhost:3000. Layar gelap dengan input company harus muncul. Tidak ada white screen / console error merah.
  • ✓
    Step 4 — Streaming timeline. Ketik Ramp → klik Analyze. AgentTimeline di kanan harus animasi step-by-step (start → setiap step running lalu done → result). Indikator "Live" muncul saat streaming aktif.
  • ✓
    Step 5 — Semua panel terisi. Setelah streaming selesai cek: WhyNowScore ring punya angka, SignalGrid menampilkan ikon-ikon signal, EvidenceLedger tabelnya berisi baris dengan source + Bright Data tool, ActionPack menampilkan cold email + LinkedIn + pertanyaan discovery.
  • ✓
    Step 6 — Mode badge. Di header cockpit harus tampak badge mock. Ini sengaja tidak disembunyikan — bagian dari safety story.
  • ✓
    Step 7 — Determinisme. Analyze Ramp dua kali. Skor & signals harus identik (mock seeded). Kalau beda → bug.
  • ✓
    Step 8 — Crash test (penting untuk demo). Matikan backend (Ctrl+C di terminal backend), refresh frontend. Cockpit harus tetap render empty state yang bersih — bukan white screen. Ini bukti mode aman.
Hero companies untuk pre-cache demo: Ramp, Snowflake, Vercel, Linear, Notion. Tes semuanya minimal sekali sebelum demo.

07 Mode mock / hybrid / live

Badge mode di cockpit menampilkan kondisi runtime.

mock default Bright Data & Claude tidak dikonfigurasi. Data 100% dari mock_data.py. Selalu jalan.
hybrid Claude only ANTHROPIC_API_KEY diisi, tapi Bright Data masih mock. Executive summary & why-now reason dioverlay Claude.
live full stack Bright Data tokens terisi + USE_MOCK=false. Signals datang dari SERP / Scraper / Unlocker beneran.

Mengaktifkan Claude (mode hybrid)

# backend/.env
ANTHROPIC_API_KEY=sk-ant-xxxxxxxx

Restart backend → badge berubah jadi hybrid. Kalau gagal API, fallback ke mock — tidak crash.

Mengaktifkan Bright Data (mode live)

# backend/.env
USE_MOCK=false
BRIGHT_DATA_API_TOKEN=xxxxxxxx
BRIGHT_DATA_SERP_ZONE=serp_api1
BRIGHT_DATA_UNLOCKER_ZONE=unblocker
BRIGHT_DATA_SCRAPER_DATASET_ID=gd_xxxxxx
Aturan keras: jangan commit .env. .env.example adalah kontrak — token asli hanya lokal.

08 Skenario Demo 3 Menit

Urutan tindakan saat live di depan juri. Latihan minimal 3x sebelum hari-H.

  1. 0:00 — Hook. "Sales tim ngabisin 30% waktu cari alasan kenapa harus contact company sekarang. Kita otomatisasi itu pakai live web data."
  2. 0:20 — Buka cockpit. Tunjuk badge mode live. Ketik Ramp. Klik Analyze.
  3. 0:30 — AgentTimeline. Sambil step jalan, narate: "SERP API menarik news, Scraper menarik LinkedIn jobs, Unlocker masuk ke blog yang JS-rendered."
  4. 1:10 — WhyNowScore. Tunjuk ring + 3 mini-score. Tekankan: "Angka ini deterministik, dihitung di scoring.py — bukan dihalusinasi LLM."
  5. 1:40 — EvidenceLedger. Klik salah satu evidence. "Setiap klaim punya source URL + Bright Data tool + confidence."
  6. 2:10 — ActionPack. Tunjuk cold email dengan personalisasi yang merujuk evidence. "Ini siap di-paste ke Outreach."
  7. 2:30 — JudgeMode. Buka panel infrastruktur. Tunjuk SERP + Scraper + Unlocker + MCP. "Empat permukaan Bright Data, satu agent."
  8. 2:50 — Close. "Mock mode selalu siap. Live mode menarik signal beneran. Why-now, evidence-first, no-BS."
Disiplin hari demo: jangan refactor besar, jangan tambah fitur baru. Cuma polish & rehearse. Pre-cache 5 hero companies. Pastikan crash test (step 8) lulus.
